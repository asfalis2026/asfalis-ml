# Auto SOS - Machine Learning Pipeline Report

This report summarizes the workflow, datasets, scripts, and outputs found in the `/Users/abhraneelkarmakar/Codes/Asfalis_ML/data_visualisation_` directory. It details how raw accelerometer data is processed, analyzed, and used to train an advanced machine learning model for automatic distress (SOS) detection.

## 1. Directory Structure

The directory is structured into three main components:
- **`new_datapoints/`**: The input directory containing the raw datasets (CSV files).
- **Python Scripts**: The core processing, exploration, and training pipeline (`step1`, `step2`, `step3`, and `visualise_acceleration`).
- **`output_images/`**: The output directory where all intermediate datasets, model artifacts, and visual insights are saved.

---

## 2. Datasets

The raw data is stored in the `new_datapoints` folder, which contains 20 CSV files. These files represent different physical activities and sensor readings (accelerometer data). 

The data is implicitly categorized based on the filename:
- **DANGER / SOS Events (Label = 1):** Keywords like `"fall"`, `"shaking"`, `"snatch"`, and `"impact"`.
  - *Examples:* `trip_and_fall_accidental_impact.csv`, `sudden_snatching.csv`, `strong_vigorous_shaking.csv`, `highheight_free_fall.csv`.
- **SAFE / Normal Activities (Label = 0):** Unmatched files representing routine user movements.
  - *Examples:* `fast_walking.csv`, `sitting_in_pocket.csv`, `running_in_pocket.csv`, `medium_walking.csv`.

---

## 3. The Pipeline (Scripts Overview)

The machine learning pipeline is broken down into four independent Python scripts:

### A. Raw Visualisation: `visualise_acceleration.py`
A stand-alone batch processing script that generates line graphs of the raw accelerometer data (X, Y, Z axes, and overall Magnitude) over time.
- **Input:** Raw CSV files from `new_datapoints/`.
- **Output:** Saves individual PNG plots for each CSV file into `output_images/`.

### B. Phase 1: `step1_data_exploration.py` (Windowing & Feature Extraction)
This script acknowledges that a single reading is not enough to predict distress. It looks at the *pattern of movement over time*.
- **Process:** 
  - Splits the continuous CSV reading into **non-overlapping 300-sample windows**.
  - Extracts descriptive statistical features for each window (mean, standard deviation, minimum, maximum, interquartile range, RMS) for the X, Y, Z axes and their Resultant Magnitude.
  - Automatically labels windows as `1` (DANGER) or `0` (SAFE) depending on file origin validation.
- **Output:** 
  - `labeled_windows.csv` - An intermediate feature table.
  - Various plots revealing the SAFE vs DANGER data distribution, such as feature correlation heatmaps and violin plots.

### C. Phase 2: `step2_feature_engineering.py` (Feature Standardisation)
This script engineers the data into the specific format expected by the backend ML models.
- **Process:** Maps the complex window-level stats into a strict **17-Feature Vector space**.
  - **Features 1-15:** Statistical summaries (mean, std, max, min, sum of squares) for X, Y, and Z.
  - **Features 16-17:** One-hot encoding for the sensor type (`is_accelerometer`=1, `is_gyroscope`=0).
- **Output:** 
  - `features.npz` (Numpy ready for fast loading).
  - `features_final.csv` and `metadata.pkl`.

### D. Phase 3: `step3_advanced_model_training.py`
The culminating script that trains, evaluates, and selects the best machine learning classifier.
- **Process:**
  - Loads vectors from `features.npz` and scales them.
  - Trains four advanced models: **Random Forest**, **XGBoost**, **LightGBM**, and **AdaBoost**.
  - Uses `class_weight` and ratio configurations to handle label imbalances.
  - Validates models against performance metrics (Accuracy, Precision, Recall, F1-Score, and ROC-AUC metrics).
  - Selects the best performing model (determined primarily by ROC-AUC score).
  - Defines variable sensitivity thresholds (Low, Medium, High) to trade off false alarm rates with miss rates.
- **Output:** 
  - The final `.pkl` model artifact containing the best algorithm (e.g. `auto_sos_model_LightGBM.pkl`).
  - Standard scaling model (`auto_sos_scaler.pkl`) and configured thresholds (`threshold_config.pkl`).
  - Comparative plots: Radar charts, multi-model ROC curves, and confusion matrices.
  - An autogenerated `ADVANCED_MODEL_REPORT.md` comparing model performances.
